# 🚀 Collection Framework Interview Questions

A curated collection of top interview questions for the Java Collection Framework, from basics to internal implementations.

---

## 1. Fundamentals (Level: Beginner)

**Q1: What is the Collection Framework?**
- A unified architecture for storing and manipulating groups of objects. It includes interfaces, implementations, and algorithms.

**Q2: Difference between `Collection` and `Collections`?**
- `Collection` is an interface. `Collections` is a utility class with static methods for manipulation.

**Q3: Why don't Collections support primitive types?**
- Because the framework was designed to hold object references. However, Java handles this automatically via **Autoboxing** (e.g., converting `int` to `Integer`).

**Q4: Difference between `List` and `Set`?**
- **List:** Insertion order preserved, duplicates allowed.
- **Set:** No order guarantee (usually), duplicates **not** allowed.

**Q5: What is the `Iterable` interface?**
- The root of the hierarchy. It allows an object to be the target of the "for-each loop" statement.

---

## 2. Lists & Sets (Level: Intermediate)

**Q1: `ArrayList` vs `LinkedList`?**
- `ArrayList`: Better for retrieval ($O(1)$) but slow for insertion/deletion in the middle ($O(n)$).
- `LinkedList`: Better for insertion/deletion at the ends ($O(1)$) but slow for retrieval ($O(n)$).

**Q2: How does `ArrayList` grow its size?**
- When full, it grows by **50%** of its current size (`oldCapacity + (oldCapacity >> 1)`).

**Q3: What are `Fail-Fast` and `Fail-Safe` iterators?**
- **Fail-Fast:** Throws `ConcurrentModificationException` if modified during iteration (e.g., `ArrayList`).
- **Fail-Safe:** Works on a clone, no exception (e.g., `CopyOnWriteArrayList`).

**Q4: Difference between `HashSet` and `TreeSet`?**
- `HashSet`: Faster ($O(1)$), uses hashing, no order.
- `TreeSet`: Slower ($O(\log n)$), uses a tree, sorted order.

**Q5: What is the `ListIterator`?**
- A powerful iterator for Lists that allows bi-directional traversal and modification of the list.

---

## 3. Maps & Advanced (Level: Expert)

**Q1: How does `HashMap` work internally?**
- It uses a technique called **Hashing**. It stores data in "Buckets." Since Java 8, if a bucket gets too large, it converts from a `LinkedList` to a `Balanced Tree` for better performance.

**Q2: Why must keys in a `HashMap` be immutable?**
- Because if a key's `hashCode()` changes after it's inserted, the map won't be able to find it again, leading to a memory leak.

**Q3: `HashMap` vs `Hashtable` vs `ConcurrentHashMap`?**
- `HashMap`: Not thread-safe, fast.
- `Hashtable`: Thread-safe (legacy), slow (method-level locking).
- `ConcurrentHashMap`: Thread-safe, high performance (bucket-level locking).

**Q4: What is `IdentityHashMap`?**
- A map that uses `==` (reference equality) instead of `.equals()` to compare keys.

**Q5: What is the `SequencedMap` interface introduced in Java 21?**
- It provides a uniform way to access the first and last elements and a reversed view of maps that have a defined order (like `LinkedHashMap`).

---

## 💡 Expert Tips for Interviews
1. **Always mention Time Complexity:** Use Big-O notation when explaining performance.
2. **Know your versions:** Mention that `HashMap` performance improved in Java 8 and `SequencedMap` was added in Java 21.
3. **Internal logic is key:** Be prepared to draw the internal structure of a `HashMap` bucket.

## One-liner to Remember
> **The Collection Framework is about choosing the right data structure for the right problem to ensure speed and scalability.**
