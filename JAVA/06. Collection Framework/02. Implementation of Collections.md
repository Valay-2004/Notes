# Implementations of Collections Framework

### List Interface

- It is the child interface of collection
- If we want to represent a group of individual objects where **duplicates are allowed** and **insertion order is preserved**. Then we should go for **List**.
- We can differentiate duplicate Objects and we can maintain insertion order by means of index hence "index plays important role in List"

Common implementations of the `List` interface include:

- `ArrayList`: Implements the `List` interface using a dynamic array, providing fast random access and efficient iteration.
- `LinkedList`: Implements the `List` interface using a doubly-linked list, allowing for efficient insertion and deletion operations.
- `Vector`: An older implementation that is synchronized, making it thread-safe. However, it is less efficient than `ArrayList`.
- `Stack`: Extends `Vector` and represents a last-in, first-out (LIFO) stack of objects.

#### 1. `ArrayList` Implementation

The `ArrayList` is the most commonly used implementation of the `List` interface. It uses a **dynamic array** to store elements.
When the array is full, it automatically grows in size. It is best for scenarios where you need fast access to elements via an index.

> [!info] Quick Summary
>
> - **Best for:** Frequent read operations and random access.
> - **Performance:** $O(1)$ for accessing elements; $O(n)$ for adding/removing from the middle.

```java
import java.util.ArrayList;
import java.util.List;

public class ArrayListExample {
    public static void main(String[] args) {
        // Creating an ArrayList of Strings
        List<String> fruits = new ArrayList<>();

        // Adding elements
        fruits.add("Apple");
        fruits.add("Banana");
        fruits.add("Mango");

        // Accessing elements by index
        System.out.println("First Fruit: " + fruits.get(0));

        // Iterating through the list
        for (String fruit : fruits) {
            System.out.println(fruit);
        }

        // Checking if the ArrayList contains a specific element
        String searchFruit = "Banana";
        if (fruits.contains(searchFruit)) {
            System.out.println(searchFruit + " is in the list.");
        } else {
            System.out.println(searchFruit + " is not in the list.");
        }

        // Removing an element from the ArrayList
        String removedFruit = "Banana";
        fruits.remove(removedFruit);
        System.out.println("After removing " + removedFruit + ":");
        for (String fruit : fruits) {
            System.out.println(fruit);
        }

        // Checking the size of the ArrayList
        System.out.println("Number of fruits in the list: " + fruits.size());

        // Clearing all elements from the ArrayList
        fruits.clear();
        System.out.println("After clearing the list, size: " + fruits.size());
    }
}
```

#### 2. `LinkedList`

A `LinkedList` implements the `List` interface using a **doubly-linked list**. Unlike `ArrayList`, it _does not_ store elements in contiguous memory. Instead, each element (node) points to the next and previous nodes.

> [!info] Quick Summary
>
> - **Best for:** Frequent additions and deletions, especially at the ends.
> - **Performance:** $O(1)$ for adding/removing at ends; $O(n)$ for accessing by index.

```java
import java.util.LinkedList;

public class LinkedListExample {
    public static void main(String[] args) {
        // Creating a LinkedList of Strings
        LinkedList<String> countries = new LinkedList<>();

        // Adding elements to the LinkedList
        countries.add("USA");
        countries.add("Canada");
        countries.add("UK");

        // Accessing elements by index
        System.out.println("First country: " + countries.getFirst());
        System.out.println("Last country: " + countries.getLast());

        // Iterating through the LinkedList
        System.out.println("All countries:");
        for (String country : countries) {
            System.out.println(country);
        }

        // Checking if the LinkedList contains a specific element
        String searchCountry = "Canada";
        if (countries.contains(searchCountry)) {
            System.out.println(searchCountry + " is in the list.");
        } else {
            System.out.println(searchCountry + " is not in the list.");
        }

        // Removing an element from the LinkedList
        String removedCountry = "Canada";
        countries.remove(removedCountry);
        System.out.println("After removing " + removedCountry + ":");
        for (String country : countries) {
            System.out.println(country);
        }

        // Checking the size of the LinkedList
        System.out.println("Number of countries in the list: " + countries.size());

        // Clearing all elements from the LinkedList
        countries.clear();
        System.out.println("After clearing the list, size: " + countries.size());
    }
}
```

#### 3. Vector

`Vector` is a legacy class similar to `ArrayList` but is **synchronized**.

> [!warning] Synchronization Overhead
> Being synchronized makes it thread-safe but significantly slower than `ArrayList`. In modern Java, use `ArrayList` unless thread-safety is strictly required.

- **Best for:** Legacy codebases or simple multi-threaded tasks.
- **Performance:** Slower due to locking overhead.

```java
import java.util.Vector;
import java.util.List;

public class VectorExample {
    public static void main(String[] args) {
        // Creating a Vector
        List<Integer> numbers = new Vector<>();

        // Adding elements
        numbers.add(10);
        numbers.add(20);
        numbers.add(30);

        // Vector is thread-safe
        System.out.println("Vector size: " + numbers.size());
        System.out.println("Element at index 1: " + numbers.get(1));
    }
}
```

#### 4. Stack

The `Stack` class extends `Vector` and follows the **LIFO (Last-In-First-Out)** principle. This means the last element added is the first one to be removed. It provides specific methods like `push()`, `pop()`, and `peek()`.

- **Best for:** Scenarios like "undo" functionality, expression evaluation, or depth-first search algorithms.

> [!NOTE] Note
> In modern Java, it is often recommended to use `Deque` (like `ArrayDeque`) for stack operations instead of `Stack` because `ArrayDeque` is faster and not synchronized.

```java
import java.util.Stack;

public class StackExample {
    public static void main(String[] args) {
        // Creating a Stack of Strings
        Stack<String> stack = new Stack<>();

        // Pushing elements onto the stack
        stack.push("Java");
        stack.push("Python");
        stack.push("C++");

        // Printing the elements in the stack
        System.out.println("Stack elements:");
        for (String element : stack) {
            System.out.println(element);
        }

        // Popping an element from the stack
        String poppedElement = stack.pop();
        System.out.println("Popped element: " + poppedElement);

        // Printing the elements after popping
        System.out.println("Stack elements after popping:");
        for (String element : stack) {
            System.out.println(element);
        }

        // Peeking at the top element without removing it
        String topElement = stack.peek();
        System.out.println("Top element without removing: " + topElement);

        // Checking if the stack is empty
        System.out.println("Is the stack empty? " + stack.isEmpty());
    }
}
```

### Cursor of Collection Framework

In the Java Collection Framework, **cursors** are objects used to traverse or access elements within a collection one by one.

> Think of them as a "pointer" that moves through your data, allowing you to interact with elements without exposing the underlying structure of the collection.

> [!abstract] Types of Cursors
>
> 1. **`Iterator`**: Universal (List, Set, Queue).
> 2. **`ListIterator`**: Powerful & Bi-directional (List only).
> 3. **`Enumeration`**: Legacy (Vector, Stack only).

#### 1. Iterator: The Universal Cursor

The `Iterator` is the most commonly used cursor. It is considered "universal" because it can be applied to any `Collection` object (List, Set, Queue, etc.). It allows for basic traversal and safe element removal during iteration.

- **Direction:** Forward only.
- **Capabilities:** Read and Remove.
- **Scope:** Any Collection.

##### Methods

- `boolean hasNext()`: Returns `true` if there are more elements.
- `Object next()`: Returns the next element.
- `void remove()`: Removes the last element returned by the iterator.

**Implementation Code**:

```java
import java.util.*;

public class IteratorExample {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>(Arrays.asList("Java", "Python", "C++", "Ruby"));

        // Creation of Iterator
        Iterator<String> it = list.iterator();

        while (it.hasNext()) {
            String language = it.next();
            if (language.equals("C++")) {
                it.remove(); // Safely removing while iterating
            } else {
                System.out.println("Language: " + language);
            }
        }
        System.out.println("Updated List: " + list);
    }
}
```

#### 2. `ListIterator`: The Powerful Bi-directional Cursor

The `ListIterator` is a specialized cursor that only works with `List` implementations. It is significantly more powerful than the standard Iterator because it allows you to move both forward and backward, and even modify the list on the fly.

- **Direction:** Bi-directional (Forward and Backward).
- **Capabilities:** Read, Remove, Add, and Replace (`set`).
- **Scope:** List implementations only (`ArrayList`, `LinkedList`, etc.).

##### Methods

- `boolean hasNext()` / `Object next()`: Standard forward traversal.
- `boolean hasPrevious()` / `Object previous()`: Backward traversal.
- `int nextIndex()` / `int previousIndex()`: Get index positions.
- `void add(Object obj)`: Inserts an element.
- `void set(Object obj)`: Replaces the last element returned.

**Implementation Code**:

```java
import java.util.*;

public class ListIteratorExample {
    public static void main(String[] args) {
        List<String> names = new ArrayList<>(Arrays.asList("Alice", "Bob", "Charlie"));

        // Creation of ListIterator
        ListIterator<String> lit = names.listIterator();

        System.out.println("Moving Forward:");
        while (lit.hasNext()) {
            String name = lit.next();
            if (name.equals("Bob")) lit.set("Bobby"); // Modifying element
            System.out.println(name);
        }

        System.out.println("\nMoving Backward:");
        while (lit.hasPrevious()) {
            System.out.println(lit.previous());
        }
    }
}
```

#### 3. Enumeration: The Legacy Cursor

`Enumeration` is the oldest cursor in Java (introduced in JDK 1.0). It is considered a "legacy" cursor because it only works with legacy classes like `Vector` and `Stack`. It is quite limited compared to modern iterators.

- **Direction:** Forward only.
- **Capabilities:** Read only (No removal or modification).
- **Scope:** Legacy classes only (`Vector`, `Stack`, `Hashtable`).

> [!NOTE] Note
> For modern applications, `Iterator` is preferred over `Enumeration` as it provides a `remove()` method and has shorter method names.

##### Methods

- `boolean hasMoreElements()`: Checks for more elements.
- `Object nextElement()`: Retrieves the next element.

**Implementation Code**:

```java
import java.util.*;

public class EnumerationExample {
    public static void main(String[] args) {
        Vector<Integer> v = new Vector<>();
        for (int i = 1; i <= 5; i++) v.add(i);

        // Creation of Enumeration using elements() method
        Enumeration<Integer> e = v.elements();

        System.out.print("Enumerating Vector: ");
        while (e.hasMoreElements()) {
            Integer i = e.nextElement();
            System.out.print(i + " ");
        }
    }
}
```

---

| **Feature**       | **Enumeration**     | **Iterator**     | **ListIterator**       |
| ----------------- | ------------------- | ---------------- | ---------------------- |
| **Legacy/Modern** | Legacy (JDK 1.0)    | Modern (JDK 1.2) | Modern (JDK 1.2)       |
| **Direction**     | Forward only        | Forward only     | Bi-directional         |
| **Accessibility** | Legacy classes only | Any Collection   | List only              |
| **Operations**    | Read only           | Read & Remove    | Read, Remove, Add, Set |

### Set Interface

#### 1. HashSet

- The `HashSet` is the go-to implementation of the `Set` interface when you need to store unique elements and don't care about the order in which they appear.
- It is powered by a **HashMap** instance internally, which is why it's so incredibly fast for basic operations.
- `HashSet` is not a synchronized class. Thus it's lightning-fast in single-threaded environments.
  - However, if we're working with multiple threads, we'll want to wrap it using `Collections.synchronizedSet()` or use a `CopyOnWriteArraySet`
- `HashSet` supports only one null value.
- `HashSet` is called unordered Collection because it is not guarantee for insertion order of elements.
- It strictly prohibits duplicate values. If you try to add an existing element, the `add()` method simply returns `false`.
- It uses a "Hashing" mechanism to calculate an index for each element, ensuring that search and retrieval times remain constant.
- It Offers $O(1)$ (constant time) performance for basic operations like add, remove, and contains, assuming the hash function disperses elements properly.

##### Constructors & Capacity

You can tune a `HashSet` using two parameters:

- **Initial Capacity** (default 16)
- **Load Factor** (default 0.75).

The load factor determines when the hash table should be resized (e.g., at 0.75, it resizes when the set is 75% full).

```java
// Default constructor (initial capacity 16, load factor 0.75)
HashSet<String> hs = new HashSet<>();

// With initial capacity
HashSet<String> hsCap = new HashSet<>(20);

// With capacity and load factor
HashSet<String> hsFull = new HashSet<>(20, 0.85f);
```

##### Implementation Code

```java
import java.util.HashSet;
import java.util.Set;

public class HashSetExample {
    public static void main(String[] args) {
        // Creating a HashSet
        Set<Integer> userIds = new HashSet<>();

        // 1. boolean add(E obj)
        userIds.add(101);
        userIds.add(102);
        userIds.add(103);

        // Attempting to add a duplicate
        boolean isAdded = userIds.add(101);
        System.out.println("Was 101 added again? " + isAdded); // Returns false

        // 2. boolean contains(E obj)
        if (userIds.contains(102)) {
            System.out.println("User 102 exists.");
        }

        // 3. int size()
        System.out.println("Total Users: " + userIds.size());

        // 4. boolean remove(E obj)
        userIds.remove(103);

        // 5. boolean isEmpty()
        System.out.println("Is set empty? " + userIds.isEmpty());

        // Printing the set (Note: Order is not guaranteed)
        System.out.println("Remaining IDs: " + userIds);

        // 6. void clear()
        userIds.clear();
        System.out.println("Size after clear: " + userIds.size());
    }
}
```

#### 2. LinkedHashSet

- `LinkedHashSet` is the implementation class of **Set interface** which is also used to store group of individual objects but **duplicates values** are not allowed.
- `LinkedHashSet` internally follows `hashtable + doubly linked list` structures.
- It is not a `synchronised` class
- It supports only **one null** value.
- It is called as **ordered Collection** because it is guarantee for insertion order.
- It provides $O(1)$ performance for `add`, `contains` and `remove`. It is slightly slower than `HashSet` because it has the extra overhead of maintaining the linked list pointers, but it is faster than `TreeSet`.
  - It consumes more memory than `HashSet` because every entry needs two additional pointers (`before and after`) to maintain the link.

###### Implementation Code

```java
import java.util.LinkedHashSet;
import java.util.Set;

public class LinkedHashSetExample {
    public static void main(String[] args) {
        // Creating a LinkedHashSet
        Set<String> registry = new LinkedHashSet<>();

        // Adding elements in a specific order
        registry.add("First User");
        registry.add("Second User");
        registry.add("Third User");

        // Attempting a duplicate
        registry.add("First User");

        // Printing the Set
        // Output will EXACTLY match insertion order: [First User, Second User, Third User]
        System.out.println("Registry Order: " + registry);

        // Common Methods
        System.out.println("Size of registry: " + registry.size());
        System.out.println("Contains 'Second User'? " + registry.contains("Second User"));

        // Removing an element
        registry.remove("Second User");
        System.out.println("After removal: " + registry);
    }
}
```

#### 3. SortedSet

- It is the child interface of `Set`
- If we want to represent a group of "unique Objects" according to some sorting order then we should go for `SortedSet interface`.
- That sorting order can be either default natural sorting order OR customized sorting order.

**`SortedSet` interface defines the following 6 specific methods**

1. Object first();
2. Object last();
3. `SortedSet` `headSet(Object o)`; It returns the elements whose elements are < o
4. `SortedSet` `tailSet(Object o)`; It returns the elements whose elements are >= o
5. `SortedSet` `subSet(Object o1,Object o2)`; It returns the elements whose elements are >=o1 and < o2
6. Comparator comparator(); Returns the comparator Object that describes under lying sorting technique. If we follow default natural sorting order then this method returns null.

```java
import java.util.SortedSet;
import java.util.TreeSet;
import java.util.Arrays;

public class SortedSetExample {
    public static void main(String[] args) {
        // Instantiate using TreeSet (the standard implementation of SortedSet)
        SortedSet<String> set = new TreeSet<>();

        // 1. Adding elements (out of order)
        set.add("Zebra");
        set.add("Apple");
        set.add("Mango");
        set.add("Banana");

        // The set automatically sorts them alphabetically
        System.out.println("Sorted Set: " + set);

        // 2. Accessing the boundaries
        System.out.println("First Element: " + set.first()); // Apple
        System.out.println("Last Element: " + set.last());   // Zebra

        // 3. Range Views (Strictly based on values, not indexes)

        // headSet: Elements strictly less than "Mango"
        SortedSet<String> beforeMango = set.headSet("Mango");
        System.out.println("HeadSet (< Mango): " + beforeMango);

        // tailSet: Elements greater than or equal to "Mango"
        SortedSet<String> fromMango = set.tailSet("Mango");
        System.out.println("TailSet (>= Mango): " + fromMango);

        // subSet: From "Banana" (inclusive) to "Zebra" (exclusive)
        SortedSet<String> middle = set.subSet("Banana", "Zebra");
        System.out.println("SubSet (Banana to Zebra): " + middle);

        // 4. Checking the Comparator
        // Returns null because we are using String's natural alphabetical order
        System.out.println("Comparator used: " + set.comparator());
    }
}
```

#### 4. TreeSet

> [!IMPORTANT] TreeSet
> While `SortedSet` provides the foundation for sorting, `TreeSet` is the actual workhorse that makes it happen.
>
> It Implements the `NavigableSet` interface, which extends `SortedSet` to provide much more granular control over _searching_ and _traversing_ the data

Internally, `TreeSet` uses a **Red-Black Tree** (a type of self-balancing binary search tree).
This ensures that no matter how many elements you add, _the set stays balanced_, and operations like _adding_, _removing_, and _searching_ always take **logarithmic time** $(O(log~ n))$.

> [!important] Key Characteristics
>
> - **Not Synchronized**: Not thread-safe.
> - **No Nulls**: Throws `NullPointerException` on adding `null`.
> - **Performance**: $O(\log n)$ — slightly slower than `HashSet` but keeps data **always sorted**.

###### Implementation Code

```java
import java.util.TreeSet;

public class TreeSetExample {
    public static void main(String[] args) {
        // Creating a TreeSet of Integers
        TreeSet<Integer> ts = new TreeSet<>();

        // Adding elements in random order
        ts.add(40);
        ts.add(10);
        ts.add(30);
        ts.add(20);
        ts.add(50);

        System.out.println("TreeSet (Sorted): " + ts); // [10, 20, 30, 40, 50]

        // --- Navigable Methods (The 'smart' search) ---

        // floor(n): Returns the greatest element <= n
        System.out.println("Floor of 25: " + ts.floor(25));   // 20

        // ceiling(n): Returns the smallest element >= n
        System.out.println("Ceiling of 25: " + ts.ceiling(25)); // 30

        // higher(n): Returns the smallest element > n
        System.out.println("Higher than 30: " + ts.higher(30)); // 40

        // lower(n): Returns the greatest element < n
        System.out.println("Lower than 30: " + ts.lower(30));   // 20

        // --- Polling (Retrieving and Removing) ---

        System.out.println("Removing Lowest: " + ts.pollFirst()); // Removes 10
        System.out.println("Removing Highest: " + ts.pollLast()); // Removes 50
        // --- Descending Order ---
        System.out.println("Reverse Order: " + ts.descendingSet());
    }
}
```

## ⚡ Edge Case Analysis

### 1. `ConcurrentModificationException`

This occurs if you try to modify a collection (add/remove) while iterating over it using a standard for-each loop or an `Iterator`.
**Solution:** Use the `Iterator.remove()` method or a `CopyOnWriteArrayList` for thread-safe concurrent modifications.

### 2. `ArrayList` Resizing Logic

When an `ArrayList` reaches its capacity, it creates a new array that is roughly **1.5 times** the size of the old one (`newCapacity = oldCapacity + (oldCapacity >> 1)`) and copies the elements over. This is why adding is $O(1)$ _amortized_ but can occasionally be $O(n)$.

### 3. Memory Overhead

- **`ArrayList`**: Minimal overhead (just the array).
- **`LinkedList`**: High overhead (each element is wrapped in a `Node` object with two pointers). Use only when frequent additions/deletions at the ends are required.

## 🙋 Interview Corner

**Q1: Difference between `ArrayList` and `Vector`?**

- `ArrayList`: Not synchronized, faster, introduced in JDK 1.2.
- `Vector`: Synchronized (thread-safe), slower, legacy class (JDK 1.0).

**Q2: Difference between `Iterator` and `ListIterator`?**

- `Iterator`: Forward only, works with all collections, can only `remove()`.
- `ListIterator`: Bi-directional, works with **List only**, can `add()`, `set()`, and `remove()`.

**Q3: What are "Fail-Fast" and "Fail-Safe" Iterators?**

- **Fail-Fast**: Throws `ConcurrentModificationException` if collection is modified during iteration (e.g., `ArrayList`, `HashSet`).
- **Fail-Safe**: Operates on a clone of the collection, so it doesn't throw an exception (e.g., `CopyOnWriteArrayList`).

**Q4: How does a `HashSet` handle duplicates internally?**
A: It uses a `HashMap` internally. When you add `element`, it calls `map.put(element, PRESENT)`. Since HashMap keys are unique, duplicates are automatically ignored.

**Q5: When should you use `TreeSet` over `HashSet`?**
A: Use `HashSet` for general-purpose high performance ($O(1)$). Use `TreeSet` only when you need the elements to be **always sorted** ($O(\log n)$).
