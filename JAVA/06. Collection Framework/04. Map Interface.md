# Map Interface: The Key-Value Powerhouse

The Java Map hierarchy has evolved significantly, specifically with the introduction of **`SequencedMap`** in Java 21.

### 1. The Map Interface

> [!Info] Map
> A Map stores data as **Key-Value pairs**. It is **not** a child of the `Collection` interface because it handles pairs rather than single objects.

> [!tip] Modern Utility Methods (Java 8 to 21+)
>
> - **`putIfAbsent(K, V)`**: Adds only if the key is missing.
> - **`getOrDefault(key, fallback)`**: Safe retrieval with a default value.
> - **`forEach((k, v) -> ...)`**: Functional iteration.
> - **`computeIfAbsent(key, function)`**: Lazy initialization.
> - **`Map.of(...)`**: Create small, unmodifiable maps instantly.

---

### 2. NEW: `SequencedMap` (Java 21+)

> [!abstract] Ordered Map Access
> Java 21 introduced **`SequencedMap`** for a uniform way to access elements in a defined order.
>
> - **Implementations**: `LinkedHashMap`, `TreeMap`.
> - **Key Methods**: `putFirst()`, `putLast()`, `firstEntry()`, `lastEntry()`, `reversed()`.

---

### 3. Implementation Classes

#### **HashMap**

- **Storage:** Uses a Hashtable (Array of Buckets).
- **Unordered:** No guarantee for insertion order.
- **Nulls:** Allows **one** null key and multiple null values.
- **Performance:** $O(1)$ for get/put. Not synchronized.
- **Internal Change:** Since Java 8, if a bucket gets too full, it switches from a LinkedList to a **Balanced Tree** to keep performance at $O(\log n)$ instead of $O(n)$.

**CODE**:

```java
import java.util.*;

public class HashMapDemo {
    public static void main(String[] args) {

        // 1. HashMap: Unordered, allows 1 null key
        Map<Integer, String> hashMap = new HashMap<>();
        hashMap.put(3, "Charlie");
        hashMap.put(1, "Alice");
        hashMap.put(2, "Bob");
        hashMap.put(null, "Ghost"); // Allowed
        System.out.println("HashMap (Unordered): " + hashMap);
    }
}
```

**Output:** `HashMap (Unordered): {null=Ghost, 1=Alice, 2=Bob, 3=Charlie}`

#### **LinkedHashMap**

- **Storage:** Hashtable + **Doubly Linked List**.
- **Ordered:** Guarantees **Insertion Order**.
- **Usage:** Best for building "LRU (Least Recently Used) Caches."

**CODE**:

```java
import java.util.*;

public class LinkedHashMapDemo {
    public static void main(String[] args) {

        // 2. LinkedHashMap: Preserves Insertion Order
        Map<Integer, String> linkedMap = new LinkedHashMap<>();
        linkedMap.put(3, "Charlie");
        linkedMap.put(1, "Alice");
        linkedMap.put(2, "Bob");
        System.out.println("LinkedHashMap (Insertion Order): " + linkedMap);
	}
}
```

**Output:** `LinkedHashMap (Insertion Order): {3=Charlie, 1=Alice, 2=Bob}`

#### **TreeMap**

- **Storage:** Red-Black Tree.
- **Sorted:** Elements are sorted by **Natural Ordering** of keys (or a custom `Comparator`).
- **Nulls:** Does **not** allow null keys (throws `NullPointerException`), but allows multiple null values.
- **Performance:** $O(\log n)$.

**CODE**:

```java
import java.util.*;

public class MapDemonstration {
    public static void main(String[] args) {

        // 3. TreeMap: Sorted by Key (Natural Order)
        Map<Integer, String> treeMap = new TreeMap<>();
        treeMap.put(3, "Charlie");
        treeMap.put(1, "Alice");
        treeMap.put(2, "Bob");
        // treeMap.put(null, "Error"); // Would throw NullPointerException
        System.out.println("TreeMap (Sorted by Key): " + treeMap);
	}
}
```

**Output:** `TreeMap (Sorted by Key): {1=Alice, 2=Bob, 3=Charlie}`

#### **Hashtable** (Legacy)

- **Synchronized:** Thread-safe but slow.
- **Nulls:** Does **not** allow null keys or null values.
- **Note:** For modern concurrent apps, use `ConcurrentHashMap` instead.

### 4. Special Purpose Maps

#### **IdentityHashMap**

- **Equality:** Uses `==` (reference equality) instead of `.equals()`.
- **Use Case:** Used during object serialization or deep cloning where you must distinguish between two identical-looking objects at different memory addresses.

**Code:**

```java
import java.util.*;
public class MapDemonstration {
    public static void main(String[] args) {

        // 4. IdentityHashMap: Comparison by reference (==)
        Map<String, String> identityMap = new IdentityHashMap<>();
        String key1 = new String("Java");
        String key2 = new String("Java");
        identityMap.put(key1, "Version 1");
        identityMap.put(key2, "Version 2");
        // Size will be 2 because key1 != key2 (different memory addresses)
        System.out.println("IdentityHashMap Size: " + identityMap.size());
    }
}
```

**Output**: `IdentityHashMap Size: 2`

#### **WeakHashMap**

- **GC Behavior:** Entry is removed if the key has no other references.
- **Use Case:** Useful for metadata or caches where you don't want the map to prevent an object from being Garbage Collected.

**Code**:

```java
import java.util.*;
public class MapDemonstration {
	public static void main(String[] args) {

        // 5. WeakHashMap: Key is eligible for GC
		Map<Object, String> weakMap = new WeakHashMap<>();
		Object key = new Object();
		weakMap.put(key, "Cache Data");
		key = null; // Removing strong reference
		System.gc(); // Suggesting Garbage Collection
		System.out.println("WeakHashMap after GC: " + weakMap);
	}
}
```

#### **Properties**

- **Type:** Keys and Values are strictly **Strings**.
- **Storage:** Extends `Hashtable`.
- **Use Case:** Storing configuration files (`.properties`).
- **Update:** Modern Java supports XML and `InputStream` loading easily.

### 5. Comparison Table:

| **Feature**      | **HashMap**   | **LinkedHashMap** | **TreeMap**          | **Hashtable**   |
| ---------------- | ------------- | ----------------- | -------------------- | --------------- |
| **Order**        | Unordered     | Insertion Order   | Sorted Order         | Unordered       |
| **Null Keys**    | Allowed (1)   | Allowed (1)       | **Not Allowed**      | **Not Allowed** |
| **Null Values**  | Allowed       | Allowed           | Allowed              | **Not Allowed** |
| **Synchronized** | No            | No                | No                   | **Yes**         |
| **Performance**  | Best ($O(1)$) | Fast ($O(1)$)     | Slower ($O(\log n)$) | Slow (Locking)  |

> [!Tip] In Multithreading
> Always prefer `Map.of()` for small, fixed maps and `ConcurrentHashMap` for multi-threaded environments. Avoid `Hashtable` in modern code.

##### Complete code with different types of Map

```java
import java.util.*;

public class MapDemonstration {
    public static void main(String[] args) {

        // 1. HashMap: Unordered, allows 1 null key
        Map<Integer, String> hashMap = new HashMap<>();
        hashMap.put(3, "Charlie");
        hashMap.put(1, "Alice");
        hashMap.put(2, "Bob");
        hashMap.put(null, "Ghost"); // Allowed
        System.out.println("HashMap (Unordered): " + hashMap);

        // 2. LinkedHashMap: Preserves Insertion Order
        Map<Integer, String> linkedMap = new LinkedHashMap<>();
        linkedMap.put(3, "Charlie");
        linkedMap.put(1, "Alice");
        linkedMap.put(2, "Bob");
        System.out.println("LinkedHashMap (Insertion Order): " + linkedMap);

        // 3. TreeMap: Sorted by Key (Natural Order)
        Map<Integer, String> treeMap = new TreeMap<>();
        treeMap.put(3, "Charlie");
        treeMap.put(1, "Alice");
        treeMap.put(2, "Bob");
        // treeMap.put(null, "Error"); // Would throw NullPointerException
        System.out.println("TreeMap (Sorted by Key): " + treeMap);

        // 4. IdentityHashMap: Comparison by reference (==)
        Map<String, String> identityMap = new IdentityHashMap<>();
        String key1 = new String("Java");
        String key2 = new String("Java");
        identityMap.put(key1, "Version 1");
        identityMap.put(key2, "Version 2");
        // Size will be 2 because key1 != key2 (different memory addresses)
        System.out.println("IdentityHashMap Size: " + identityMap.size());

        // 5. WeakHashMap: Key is eligible for GC
        Map<Object, String> weakMap = new WeakHashMap<>();
        Object key = new Object();
        weakMap.put(key, "Cache Data");
        key = null; // Removing strong reference
        System.gc(); // Suggesting Garbage Collection
        System.out.println("WeakHashMap after GC: " + weakMap);

        // 6. Java 21 SequencedMap Methods
        // Works with LinkedHashMap or TreeMap
        SequencedMap<Integer, String> seqMap = new LinkedHashMap<>();
        seqMap.putLast(1, "Start");
        seqMap.putLast(2, "End");
        seqMap.putFirst(0, "New Start");

        System.out.println("SequencedMap: " + seqMap);
        System.out.println("Reversed View: " + seqMap.reversed());
    }
}
```

##### Modern Java "Shortcuts" (Java 9+)

When you need an immutable map for quick lookup, use the static factory methods:

```java
// Create a fixed-size, unmodifiable map
Map<String, Integer> scores = Map.of(
    "Alice", 90,
    "Bob", 85,
    "Charlie", 88
);

// This will throw UnsupportedOperationException
// scores.put("Dave", 70);
```

> [!success] Quick Reference
>
> - **`HashMap`**: The standard choice. Use unless order is needed.
> - **`LinkedHashMap`**: Use when insertion order matters or for LRU caches.
> - **`TreeMap`**: Use when keys must be sorted naturally.
> - **`ConcurrentHashMap`**: The modern choice for multi-threaded apps.
> - **`Hashtable` / `Vector`**: Legacy. Avoid. Use `ConcurrentHashMap` for threads.
> - **`IdentityHashMap`**: Rare. Use only when you need to distinguish between different objects with the same content.

## ⚡ Edge Case Analysis

### 1. Hash Collision Performance

If many keys have the same `hashCode()`, they fall into the same bucket.

- **Before Java 8:** Bucket used a `LinkedList`, retrieval was $O(n)$.
- **Since Java 8:** If a bucket exceeds a threshold (8 elements), it converts to a **Balanced Tree**, bringing performance back to $O(\log n)$.

### 2. Mutable Keys in HashMap

If you use a mutable object (like a custom `Student` object) as a Key and then change its fields, its `hashCode()` will change. The Map will be unable to find the entry again, leading to a memory leak.
**Solution:** Always use **Immutable** objects as keys (e.g., `String`, `Integer`).

## 🙋 Interview Corner

**Q1: How does `HashMap` work internally? (The Million Dollar Question)**

1. It uses `hashCode()` of the key to find the bucket index.
2. If multiple keys have the same index (Collision), it uses `equals()` to find the correct entry in the bucket's list/tree.

**Q2: Difference between `HashMap` and `ConcurrentHashMap`?**

- `HashMap`: Not synchronized, faster, allows 1 null key.
- `ConcurrentHashMap`: Thread-safe, faster than `Hashtable` (uses segment/bucket-level locking), does **not** allow null keys or values.

**Q3: What is the significance of `Load Factor` (default 0.75)?**
A: It is the measure that determines when to double the capacity of the map. If the map is 75% full, it rehashes all elements into a larger array.

**Q4: Difference between `TreeMap` and `LinkedHashMap`?**

- `TreeMap`: Sorted by **Natural Order** of keys ($O(\log n)$).
- `LinkedHashMap`: Preserves **Insertion Order** ($O(1)$).
