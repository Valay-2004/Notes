# Practice Problems: Time & Space Complexity

Here are some practice problems to help you master analyzing the time and space complexity of algorithms. These examples are now implemented in **Java** and include detailed explanations for key logic.

---

## 1. Linear Search

**Problem:** Given an array of integers and a target value, find the index of the target. If not present, return -1.

```java
public int linearSearch(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) {
            return i; // Target found
        }
    }
    return -1; // Target not found
}
```

- **Time Complexity:** $O(n)$ — In the worst case, we check every element.
- **Space Complexity:** $O(1)$ — No extra space proportional to input size.

## 2. Binary Search

**Problem:** Given a **sorted** array, find the index of the target value.

```java
public int binarySearch(int[] arr, int target) {
    int left = 0, right = arr.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) {
            return mid;
        }
        if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return -1;
}
```

> [!INFO] Detailed Explanation
>
> 1. **Divide and Conquer:** In each iteration, we compare the `target` with the middle element (`mid`).
> 2. **Elimination:** If `target` is smaller, we know it must be in the left half, so we ignore the right half. If larger, we ignore the left.
> 3. **Logarithmic Growth:** Because the search space is halved every time ($n \rightarrow n/2 \rightarrow n/4 \dots$), it takes $\log_2 n$ steps to reach a single element.
> 4. **Overflow Prevention:** Using `left + (right - left) / 2` instead of `(left + right) / 2` prevents potential integer overflow when `left` and `right` are large.

- **Time Complexity:** $O(\log n)$
- **Space Complexity:** $O(1)$

## 3. Find Maximum in an Array

**Problem:** Find the maximum element in an integer array.

```java
public int findMax(int[] arr) {
    int maxVal = arr[0];
    for (int i = 1; i < arr.length; i++) {
        if (arr[i] > maxVal) {
            maxVal = arr[i];
        }
    }
    return maxVal;
}
```

- **Time Complexity:** $O(n)$
- **Space Complexity:** $O(1)$

## 4. Sum of All Subsets

**Problem:** Find the sum of all possible subsets of an array.

```java
public int sumOfSubsets(int[] arr) {
    int n = arr.length;
    int totalSum = 0;
    int numOfSubsets = 1 << n; // 2^n subsets

    for (int i = 0; i < numOfSubsets; i++) {
        int subsetSum = 0;
        for (int j = 0; j < n; j++) {
            if ((i & (1 << j)) != 0) {
                subsetSum += arr[j];
            }
        }
        totalSum += subsetSum;
    }
    return totalSum;
}
```

> [!INFO] Detailed Explanation
>
> 1. **Subset Count:** An array of size $n$ has $2^{n}$ subsets (including the empty set).
> 2. **Bitmasking:** We use an integer `i` from `0` to `2^n - 1` as a bitmask. If the $j$-th bit of `i` is set, the $j$-th element of the array is included in the current subset.
> 3. **Nested Loops:** The outer loop runs $2^{n}$ times, and the inner loop runs $n$ times.
> 4. **Total Work:** Total operations = $n \times 2^{n}$.

- **Time Complexity:** $O(n \cdot 2^{n})$
- **Space Complexity:** $O(1)$

## 5. Bubble Sort

**Problem:** Sort an array using the Bubble Sort algorithm.

```java
public void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                // Swap
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}
```

- **Time Complexity:** $O(n^{2})$
- **Space Complexity:** $O(1)$

## 6. Fibonacci Sequence (Recursive)

**Problem:** Find the $n$-th Fibonacci number recursively.

```java
public int fibonacci(int n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}
```

- **Time Complexity:** $O(2^{n})$ — Each call branches into two more calls.
- **Space Complexity:** $O(n)$ — Depth of the recursion stack.

## 7. Fibonacci Sequence (Dynamic Programming)

**Problem:** Optimized Fibonacci using an array.

```java
public int fibonacciDP(int n) {
    if (n <= 1) return n;
    int[] fib = new int[n + 1];
    fib[0] = 0;
    fib[1] = 1;
    for (int i = 2; i <= n; i++) {
        fib[i] = fib[i - 1] + fib[i - 2];
    }
    return fib[n];
}
```

- **Time Complexity:** $O(n)$ — We compute each value exactly once.
- **Space Complexity:** $O(n)$ — To store the `fib` array.

## 8. Find the First Non-Repeating Character

**Problem:** Given a string, find the first character that does not repeat.

```java
public char firstNonRepeatingChar(String s) {
    int[] charCount = new int[256]; // Assuming ASCII
    for (int i = 0; i < s.length(); i++) {
        charCount[s.charAt(i)]++;
    }
    for (int i = 0; i < s.length(); i++) {
        if (charCount[s.charAt(i)] == 1) {
            return s.charAt(i);
        }
    }
    return '\0';
}
```

- **Time Complexity:** $O(n)$ — Two passes over the string.
- **Space Complexity:** $O(1)$ — The array size (256) is constant regardless of input string length.

## 9. Reverse a Linked List

**Problem:** Reverse a singly linked list.

```java
class Node {
    int data;
    Node next;
}

public Node reverseLinkedList(Node head) {
    Node prev = null;
    Node current = head;
    Node next = null;
    while (current != null) {
        next = current.next;
        current.next = prev;
        prev = current;
        current = next;
    }
    return prev;
}
```

- **Time Complexity:** $O(n)$
- **Space Complexity:** $O(1)$

## 10. Merge Two Sorted Arrays

**Problem:** Merge two sorted arrays into a new sorted array.

```java
public int[] mergeArrays(int[] arr1, int[] arr2) {
    int n1 = arr1.length, n2 = arr2.length;
    int[] result = new int[n1 + n2];
    int i = 0, j = 0, k = 0;

    while (i < n1 && j < n2) {
        if (arr1[i] < arr2[j]) result[k++] = arr1[i++];
        else result[k++] = arr2[j++];
    }
    while (i < n1) result[k++] = arr1[i++];
    while (j < n2) result[k++] = arr2[j++];

    return result;
}
```

- **Time Complexity:** $O(n_1 + n_2)$
- **Space Complexity:** $O(n_1 + n_2)$

## 11. Find All Permutations of a String

**Problem:** Generate all permutations of a string.

```java
public void permute(char[] arr, int l, int r) {
    if (l == r) {
        System.out.println(String.valueOf(arr));
    } else {
        for (int i = l; i <= r; i++) {
            swap(arr, l, i);
            permute(arr, l + 1, r);
            swap(arr, l, i); // Backtrack
        }
    }
}

private void swap(char[] arr, int i, int j) {
    char temp = arr[i];
    arr[i] = arr[j];
    arr[j] = temp;
}
```

- **Time Complexity:** $O(n!)$ — There are $n!$ permutations.
- **Space Complexity:** $O(n)$ — Stack depth.

## 12. Count the Number of Set Bits

**Problem:** Count 1s in the binary representation of an integer.

```java
public int countSetBits(int n) {
    int count = 0;
    while (n > 0) {
        count += n & 1;
        n >>= 1;
    }
    return count;
}
```

- **Time Complexity:** $O(\log n)$ — Number of bits in $n$ is $\log_2 n$.
- **Space Complexity:** $O(1)$

## 13. Check if a String is a Palindrome

**Problem:** Check if a string reads the same forwards and backwards.

```java
public boolean isPalindrome(String s) {
    int left = 0, right = s.length() - 1;
    while (left < right) {
        if (s.charAt(left) != s.charAt(right)) return false;
        left++;
        right--;
    }
    return true;
}
```

- **Time Complexity:** $O(n)$
- **Space Complexity:** $O(1)$

## 14. Find the Intersection of Two Arrays

**Problem:** Find common elements between two arrays.

```java
import java.util.*;

public List<Integer> intersection(int[] arr1, int[] arr2) {
    Set<Integer> set = new HashSet<>();
    for (int val : arr1) set.add(val);

    List<Integer> result = new ArrayList<>();
    for (int val : arr2) {
        if (set.contains(val)) {
            result.add(val);
            set.remove(val); // Unique elements only
        }
    }
    return result;
}
```

- **Time Complexity:** $O(n_1 + n_2)$ — Set operations are $O(1)$ on average.
- **Space Complexity:** $O(n_1)$ — To store the first array in a Set.

## 15. Rotate an Array

**Problem:** Rotate an array by $k$ positions to the right.

```java
public void rotateArray(int[] arr, int k) {
    int n = arr.length;
    k = k % n;
    reverse(arr, 0, n - 1);
    reverse(arr, 0, k - 1);
    reverse(arr, k, n - 1);
}

private void reverse(int[] arr, int start, int end) {
    while (start < end) {
        int temp = arr[start];
        arr[start] = arr[end];
        arr[end] = temp;
        start++;
        end--;
    }
}
```

- **Time Complexity:** $O(n)$ — Three passes over the array.
- **Space Complexity:** $O(1)$

# Edge Case Analysis

When analyzing complexity and writing robust code, always consider these edge cases:

### 1. Empty or Null Input

- **Scenario:** An array of size 0 or a null reference.
- **Impact:** Many algorithms (like `findMax`) will crash if they assume at least one element exists (`arr[0]`).
- **Fix:** Add a check: `if (arr == null || arr.length == 0) return ...;`

### 2. Single Element Input

- **Scenario:** Array has exactly one element.
- **Complexity:** Usually $O(1)$ for that specific case, but asymptotic complexity remains the same.
- **Check:** Does your `while (left <= right)` or `for` loop handle $n=1$ correctly?

### 3. Already Sorted vs. Reverse Sorted

- **Scenario:** For sorting algorithms, the initial state matters.
- **Example:** **QuickSort** can degrade to $O(n^{2})$ on a sorted array if the pivot selection is poor. **Bubble Sort** can be optimized to $O(n)$ for a sorted array using a "swapped" flag.

### 4. All Elements are the Same

- **Scenario:** `[5, 5, 5, 5]`.
- **Complexity:** Some algorithms like QuickSort can have issues with partitions if not handled (Dutch National Flag algorithm).

### 5. Integer Overflow

- **Scenario:** Calculating the middle index in Binary Search.
- **Problem:** `(left + right) / 2` can exceed $2^{31}-1$ if `left` and `right` are large.
- **Solution:** Use `left + (right - left) / 2`.

> [!TIP] Pro Tip
> In interviews, always mention these edge cases _before_ writing the code. It shows you think about "worst-case" scenarios beyond just the input size $n$.
