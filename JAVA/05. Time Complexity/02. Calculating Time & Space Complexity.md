# Calculate Time and Space Complexity

## Step 1: Understand the Code

> [!info] Purpose  
> Before analyzing complexity, you must clearly understand what the code is doing.

Carefully read the code and identify:

- What problem it is solving
- The overall flow of execution
- Important components such as:
  - Loops
  - Recursive calls
  - Data structures

> [!tip] Practical Thinking
> Try to identify which part of the code runs the greatest number of times. That part usually dominates the time complexity.

---

## Step 2: Identify Key Operations

> [!important] Core idea  
> Time complexity depends on how many times key operations are executed.

### Loops

Determine:

- How many times each loop runs
- Whether loops are independent or nested

### Conditionals

Check:

- Which branch runs more frequently
- Always consider the branch with the highest cost (worst case)

### Recursion

Understand:

- How many recursive calls are made
- How deep the recursion goes

### Data Structure Operations

Look for operations like:

- Access
- Insertion
- Deletion

Each data structure has different costs for these operations.

---

## Step 3: Calculate Time Complexity

### Constant Time – $O(1)$

> [!note] Definition  
> The execution time does not depend on input size.

Example:

```java
int x = 5;
```

This operation always takes the same amount of time.

---

### Single Loop – $O(n)$

> [!note] Explanation  
> A loop running from 0 to n executes n times.

```java
for (int i = 0; i < n; i++) {
    // constant work
}
```

Each iteration does constant work, so total complexity becomes $O(n)$.

---

### Nested Loops – $O(n^{2})$

> [!warning] Key rule  
> When loops are nested, multiply their complexities.

```java
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        // constant work
    }
}
```

Outer loop runs $n$ times and inner loop runs $n$ times for each iteration.  
Total operations = $n \times n = O(n^{2})$.

---

### Conditionals

> [!important] Rule  
> Always consider the branch with the highest time complexity.

```java
if (condition) {
    // O(1)
} else {
    // O(n)
}
```

Even if the $O(n)$ branch runs less frequently, worst-case complexity is $O(n)$.

---

### Recursion

> [!note] Approach  
> Express recursion using a recurrence relation.

Example:

```java
int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}
```

- Recurrence: $T(n) = T(n-1) + T(n-2)$
- This results in exponential growth.
- Final complexity: $O(2^{n})$

> [!tip] Pattern recognition
>
> - One recursive call → often $O(n)$
> - Two recursive calls → often exponential

---

### Data Structure Operations

> [!info] Important reference

| Operation                | Complexity |
| ------------------------ | ---------- |
| Array access             | $O(1)$     |
| Array insertion/deletion | $O(n)$     |
| HashMap access           | $O(1)$     |
| HashMap insertion        | $O(1)$     |

Understanding these helps in analyzing real-world problems.

---

## Step 4: Calculate Space Complexity

### Variables – $O(1)$

> [!note] Explanation  
> Simple variables take constant space.

```java
int x;
```

---

### Arrays and Containers

- One-dimensional array → $O(n)$
- Two-dimensional array → $O(n × m)$

```java
int arr[n];      // O(n)
int arr[n][m];   // O(n * m)
```

---

### Auxiliary Space

> [!note] Definition  
> Extra space used by the algorithm apart from input.

Example:

```java
int sum = 0;
```

This uses constant space → $O(1)$

---

### Recursive Call Stack

> [!warning] Important concept  
> Each recursive call uses stack memory.

```java
int factorial(int n) {
    if (n == 0) return 1;
    return n * factorial(n - 1);
}
```

- Number of calls = n
- Stack space = $O(n)$

---

## Step 5: Simplify Big-O Expression

> [!important] Simplification rules

1. Ignore constants
2. Ignore lower-order terms
3. Keep only the dominant term

Examples:

- $O(2n + 5) \rightarrow O(n)$
- $O(n^{2} + n) \rightarrow O(n^{2})$

> [!tip] Key idea  
> Focus on how the algorithm behaves for large inputs.

---

## Step 6: Best, Worst, and Average Case

### Best Case

- Minimum number of operations
- Rarely used in analysis

### Worst Case

- Maximum number of operations
- Most important for interviews

### Average Case

- Expected performance over all inputs

> [!important] Interview rule  
> Always mention worst-case complexity unless asked otherwise.

---

## Step 7: Practice Examples

---

### Example 1: Simple Loop

```java
for (int i = 0; i < n; i++) {
}
```

- Time Complexity: $O(n)$
- Space Complexity: $O(1)$

---

### Example 2: Nested Loop

```java
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
    }
}
```

- Time Complexity: $O(n²)$
- Space Complexity: $O(1)$

---

### Example 3: Binary Search

```java
while (left <= right) {
    int mid = left + (right - left) / 2;
}
```

- Time Complexity: $O(\log n)$
- Space Complexity: $O(1)$

> [!note] Reason  
> The search space is reduced by half in each step.

---

### Example 4: Merge Two Sorted Arrays

```java
while (i < n1 && j < n2) {
    // merge logic
}
```

- Time Complexity: $O(n_1 + n_2)$
- Space Complexity: $O(n_1 + n_2)$

---

## Final Tips

> [!success] Practical advice
>
> - Break the code into smaller parts before analyzing
> - Focus on loops, recursion, and data structure operations
> - Always calculate worst-case complexity
> - Practice regularly to recognize patterns quickly

> [!quote] Key takeaway  
> Time complexity is determined by how many times the main operation runs as the input size increases.
