# Collection Framework

## Comparable & Comparator

> More detailed article about this [blog.ashutoshkrris.in](https://blog.ashutoshkrris.in/comparable-vs-comparator-explained-in-java).

Sorting is a core programming task. While Java's `Arrays.sort()` and `Collections.sort()` easily handle primitives and strings, they cannot automatically order custom objects.

To sort user-defined classes, developers must implement the `Comparable` or `Comparable` interfaces to define a custom sorting logic tailored to specific requirements.

### Sorting Methods for Primitive Types

Java provides a variety of `built-in` sorting methods that can be used to sort primitive data types which makes sorting easy.
These methods are highly optimized and efficient, which leads us to sort array with minimal code.
For primitive types, such as integers, floating-point numbers, and characters, the `Arrays.sort()` method is commonly used.

#### Arrays.sort()

The `Arrays.sort()` method sorts the specified array into ascending numerical order. This method uses a `Dual-Pivot Quicksort` algorithm, which is faster and more efficient for most data sets.

**Example**:

```java
package tutorial;

import java.util.Arrays;

public class PrimitiveSorting {
    public static void main(String[] args) {
        int[] numbers = { 5, 3, 8, 2, 1 };
        System.out.println("Original array: " + Arrays.toString(numbers));

        Arrays.sort(numbers);
        System.out.println("Sorted array: " + Arrays.toString(numbers));

        char[] characters = { 'o', 'i', 'e', 'u', 'a' };
        System.out.println("Original array: " + Arrays.toString(characters));

        Arrays.sort(characters);
        System.out.println("Sorted array: " + Arrays.toString(characters));
    }
}
```

**Output**:

```
Original array: [5, 3, 8, 2, 1]
Sorted array: [1, 2, 3, 5, 8]
Original array: [o, i, e, u, a]
Sorted array: [a, e, i, o, u]
```

#### Collections.sort()

The `Collections.sort()` method is used to sort collections such as `ArrayList`.
This method is also based on the natural ordering of the elements or a `custom` comparator.

```java
package tutorial;

import java.util.ArrayList;
import java.util.Collections;

public class CollectionsSorting {
    public static void main(String[] args) {
        ArrayList<String> wordsList = new ArrayList<>();
        wordsList.add("banana");
        wordsList.add("apple");
        wordsList.add("cherry");
        wordsList.add("date");
        System.out.println("Original list: " + wordsList);

        Collections.sort(wordsList);
        System.out.println("Sorted list: " + wordsList);
    }
}
```

Output:

```
Original list: [banana, apple, cherry, date]
Sorted list: [apple, banana, cherry, date]
```

##### Limitation with Custom Classes

Standard sorting methods like `Arrays.sort()` and `Collections.sort()` work seamlessly for primitives and types with **natural ordering** (e.g., `String`, `Integer`). However, they fail with custom objects because Java doesn't know which attribute to prioritize for comparison.

###### The Problem: Lack of Comparison Logic

If you attempt to sort a list of custom objects without defining a comparison rule, the compiler throws a **`ClassCastException`** or a **Compilation Error**.

```java
package tutorial;

public class Person {
    String name;
    int age;
    double weight;

    public Person(String name, int age, double weight) {
        this.name = name;
        this.age = age;
        this.weight = weight;
    }

    @Override
    public String toString() {
        return "Person [name=" + name + ", age=" + age + ", weight=" + weight + " kgs]";
    }
}

```

The `sort()` method requires the objects to either implement the `Comparable` interface or be accompanied by a `Comparator`. Without these, Java cannot determine if `Person A` is "less than" or "greater than" `Person B`.

To sort custom objects like `Person`, we need to provide a way to compare these objects. Java offers two main approaches to achieve this:

1. Implementing the `Comparable` Interface: This allows a class to define its natural ordering by implementing the `compareTo` method.
2. Using the `Comparator` Interface: This allows us to create separate classes or `lambda` expressions to define multiple ways of comparing objects.

### Comparable Interface

> [!IMPORTANT] Comparable Interface
> Java provides a `Comparable Interface` to define a natural ordering for objects of a user-defined class.

By implementing the `comparable` interface we can get a single natural ordering that can be used to sort its instance. This is particularly useful when you need a default way to compare and sort objects.

#### Overview

The **`Comparable<T>`** interface allows a class to define its own default sorting logic, known as **Natural Ordering**. By implementing this interface, objects of that class can be sorted automatically by `Collections.sort()` or `Arrays.sort()`.

> [!info] The `compareTo()` Method
> This is the single method required. It compares the current object (`this`) with a specified object (`other`).
>
> | **Return Value** | **Meaning**       |
> | :--------------- | :---------------- |
> | **Negative**     | `this` < `other`  |
> | **Zero**         | `this` == `other` |
> | **Positive**     | `this` > `other`  |

**Implementation Example:**

```java
public class PersonV2 implements Comparable<PersonV2> {
    String name;
    int age;
    double weight;

    public PersonV2(String name, int age, double weight) {
        this.name = name;
        this.age = age;
        this.weight = weight;
    }

    @Override
    public int compareTo(PersonV2 other) {
        // Option 1: Manual subtraction (Common for primitives)
        // return this.age - other.age;

        // Option 2: Using Helper Method (Recommended to avoid overflow)
        return Integer.compare(this.age, other.age);
    }

    @Override
    public String toString() {
        return name + " (" + age + ")";
    }
}
```

**Usage:**
Once `Comparable` is implemented, sorting is straightforward:

```java
List<PersonV2> people = new ArrayList<>(Arrays.asList(
    new PersonV2("Alice", 30, 65.5),
    new PersonV2("Bob", 25, 75.0),
    new PersonV2("Charlie", 35, 80.0)
));

Collections.sort(people); // Uses the compareTo method inside PersonV2
```

**Output:** `[Bob (25), Alice (30), Charlie (35)]`

> In this example, the `PersonV2` objects are sorted in ascending order of age using the `Collections.sort()` method, which relies on the natural ordering defined by the `compareTo()` method in the `PersonV2` class.

> [!warning] Limitations of `Comparable`
>
> 1. **Single Ordering**: Only one sort order per class.
> 2. **Modification Required**: You must modify the original source code.
> 3. **Rigidity**: Changing logic might break other parts of the app.

> [!TIP] TIP
> To solve these issues specifically to allow multiple sorting criteria (by `name`, then by `weight`, etc.) → Java provides the **`Comparator`** interface.

### Comparator Interface

> [!IMPORTANT] Comparator Interface
> The `Comparator` interface in Java provides a way to define multiple ways to compare and sort objects. Unlike the `Comparable` interface, which allows only a single natural ordering, `Comparator` is designed to offer flexibility by allowing multiple sorting strategies. This makes it particularly useful for scenarios where objects need to be sorted in different ways.

#### Overview

Unlike `Comparable`, the **`Comparator<T>`** interface is defined **outside** the class being sorted. This allows you to create multiple sorting strategies for the same object without modifying the original class.

- **Multiple Strategies:** You can have one comparator for "Name," another for "Age," etc.
- **Decoupled:** The sorting logic is separate from the data class.
- **Functional:** It is a functional interface, meaning it can be used with **Lambda Expressions**.

> [!info] The `compare()` Method
> Compares two separate objects: `compare(T o1, T o2)`.
>
> - **Negative**: `o1` < `o2`
> - **Zero**: `o1` == `o2`
> - **Positive**: `o1` > `o2`

**Implementation Example:**
By creating dedicated classes, you can swap sorting logic at runtime.

**Updated `Person` class**:

```java
package tutorial;

public class Person {
    String name;
    int age;
    double weight;

    public Person(String name, int age, double weight) {
        this.name = name;
        this.age = age;
        this.weight = weight;
    }

    public String getName() {
        return name;
    }

    public int getAge() {
        return age;
    }

    public double getWeight() {
        return weight;
    }

    @Override
    public String toString() {
        return "Person [name=" + name + ", age=" + age + ", weight=" + weight + " kgs]";
    }
}

```

1. **Comparator by Name** (Sort by Name).
   This comparator sorts `Person` objects alphabetically by their `name`.

   ```java
   public class PersonNameComparator implements Comparator<Person> {
      @Override
      public int compare(Person p1, Person p2){
   	   return p1.getName().compareTo(p2.getName());
      }
   }
   ```

2. **Comparator by Age** (Sort by Age).
   This comparator sorts `Person` objects by their `age`, in ascending order.

   ```java
   public class PersonAgeComparator implements Comparator<Person> {
     @Override
     public int compare(Person p1, Person p2) {
         // return p1.getAge() - p2.getAge();
         return Integer.compare(p1.getAge(), p2.getAge());
     }
   }
   ```

3. `Comparator by Weight` (Sort by Weight).
   This comparator sorts `Person` objects by their `weight`, in ascending order.
   ```java
   public class PersonWeightComparator implements Comparator<Person> {
      @Override
      public int compare(Person p1, Person p2) {
          return Double.compare(p1.getWeight(), p2.getWeight());
      }
   }
   ```

**Usage Showing code:**

```java
package tutorial;

import tutorial.comparator.PersonAgeComparator;
import tutorial.comparator.PersonNameComparator;
import tutorial.comparator.PersonWeightComparator;

import java.util.ArrayList;
import java.util.Arrays;
import java.util.Collections;
import java.util.List;

public class CustomClassSortingV3 {
    public static void main(String[] args) {
        List<Person> people = new ArrayList<>(Arrays.asList(
                new Person("Alice", 30, 65.5),
                new Person("Bob", 25, 75.0),
                new Person("Charlie", 35, 80.0)
        ));
        System.out.println("Original people list: " + people);

        Collections.sort(people, new PersonNameComparator());
        System.out.println("Sorted people list by name: " + people);

        Collections.sort(people, new PersonAgeComparator());
        System.out.println("Sorted people list by age: " + people);

        Collections.sort(people, new PersonWeightComparator());
        System.out.println("Sorted people list by weight: " + people);
    }
}
```

**Output:**

```plain
Original people list: [Person [name=Alice, age=30, weight=65.5 kgs], Person [name=Bob, age=25, weight=75.0 kgs], Person [name=Charlie, age=35, weight=80.0 kgs]]
Sorted people list by name: [Person [name=Alice, age=30, weight=65.5 kgs], Person [name=Bob, age=25, weight=75.0 kgs], Person [name=Charlie, age=35, weight=80.0 kgs]]
Sorted people list by age: [Person [name=Bob, age=25, weight=75.0 kgs], Person [name=Alice, age=30, weight=65.5 kgs], Person [name=Charlie, age=35, weight=80.0 kgs]]
Sorted people list by weight: [Person [name=Alice, age=30, weight=65.5 kgs], Person [name=Bob, age=25, weight=75.0 kgs], Person [name=Charlie, age=35, weight=80.0 kgs]]
```

In this example, the `Comparator` instances allow sorting the `Person` objects by different attributes: name, age, and weight. This demonstrates how the `Comparator` interface enables flexible and versatile sorting strategies for a class.

### Comparable vs Comparator

> [!Info] Comparable vs Comparator
> When sorting objects in Java, you have two primary options: the `Comparable` and `Comparator` interfaces. Understanding the differences between these two interfaces can help you choose the right approach for your needs. Please note that this is also a **very important interview question**.

| Feature            | Comparable                                                                  | Comparator                                                                       |
| ------------------ | --------------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| Definition         | Provides a single, natural ordering for objects                             | Provides multiple ways to compare objects                                        |
| Method             | compareTo(T o)                                                              | compare(T o1, T o2)                                                              |
| Implementation     | Implemented within the class itself                                         | Implemented outside the class                                                    |
| Sorting Criteria   | One default natural ordering                                                | Multiple sorting criteria                                                        |
| Flexibility        | Limited to one way of comparing objects                                     | Flexible; multiple comparators can be defined                                    |
| Class Modification | Requires modifying the class to implement `Comparable`                      | Does not require modifying the class                                             |
| Use Case           | Use when there is a clear, natural ordering (e.g., sorting employees by ID) | Use when different sorting orders are needed or when you cannot modify the class |

#### Summary: Comparable vs. Comparator

Choosing between `Comparable` and `Comparator` depends on whether you need a permanent, "default" sort order or the flexibility to sort by different fields at different times.

---

##### **Comparable**

- **Benefits:**
  - **Simplicity:** Once implemented, `Collections.sort(list)` just works without extra arguments.
  - **Standardization:** Defines the "natural" identity of the object's order.
- **Drawbacks:**
  - **Rigidity:** You are locked into one sorting logic (e.g., if you sort by `ID`, you can't easily switch to `Name` using this interface).
  - **Intrusive:** You must modify the class code to implement it.

##### **Comparator**

- **Benefits:**
  - **Extreme Flexibility:** You can define infinite ways to sort (Ascending, Descending, by different fields).
  - **Clean Design:** Keeps sorting logic separate from the data model (Single Responsibility Principle).
- **Drawbacks:**
  - **Management:** Requires passing the comparator every time you sort.
  - **Slight Overhead:** Creating many anonymous classes or instances can slightly increase memory usage (though modern Lambdas mitigate this).

---

> [!success] Final Verdict: Which to use?
>
> - Use **`Comparable`** for the logical, "natural" way to order a class (e.g., sorting employees by ID).
> - Use **`Comparator`** when you need the flexibility to sort in different ways (by Name, then by Date, etc.) or when you cannot modify the original class code.
