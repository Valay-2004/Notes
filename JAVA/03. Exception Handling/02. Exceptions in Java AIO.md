# Exception Handling in Java - Complete Revision Notes

> [!abstract] Why Exception Handling?
> Exception Handling allows us to write **robust** programs that don't crash on unexpected situations.  
> Instead of abrupt termination, we can show friendly messages and continue execution.

## What is an Exception?

An **exception** is an **abnormal event** or **runtime error** that disrupts the normal flow of a program.

**Real-life Analogy**:  
You are driving smoothly (normal flow). Suddenly a dog appears on the road → this is an **exception**.

- Without handling → car crashes (program terminates).
- With handling → you apply brakes, avoid crash, and continue driving.

**Examples of Exceptions**:

- Dividing a number by zero (`10 / 0`)
- Accessing a null object (`null.toString()`)
- Trying to read a file that doesn't exist
- Entering wrong data type (String instead of int)

**Exception vs Error**:

- **Exception** → Recoverable (we can handle it).
- **Error** → Serious JVM issues (e.g., `OutOfMemoryError`, `StackOverflowError`) → usually **not handled** by programmers.

## try-catch Block

The most basic way to handle exceptions.

- **`try`** → Put risky code here.
- **`catch`** → Handle the exception if it occurs.

```java
import java.util.Scanner;

public class TryCatchDemo {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        try {
            System.out.print("Enter a number: ");
            int num = sc.nextInt();                    // Risky line
            System.out.println("You entered: " + num);
        }
        catch (Exception e) {                          // Catches any exception
            System.out.println("Something went wrong: " + e.getMessage());
            // e.printStackTrace();                    // Optional: full error details
        }

        System.out.println("Program continues after handling...");
    }
}
```

**How it works**:

- If no exception → `try` runs fully, `catch` is skipped.
- If exception occurs → control jumps to `catch`, `try` stops at the error line.
- After `catch`, the rest of the program continues.

## Types of Exceptions - Checked vs Unchecked

### 1. Checked Exceptions (Compile-time Exceptions)

- Compiler **forces** you to handle them.
- Occur due to external factors (file not found, network issues, etc.).
- Examples: `IOException`, `SQLException`, `ClassNotFoundException`.

### 2. Unchecked Exceptions (Runtime Exceptions)

- Compiler does **not** force handling.
- Occur due to programming/logic mistakes.
- All inherit from `RuntimeException`.
- Examples: `ArithmeticException`, `NullPointerException`, `ArrayIndexOutOfBoundsException`, `InputMismatchException`.

**Compiler Behavior**:

- **Checked** → Compiler checks at compile time → error if not handled/declared.
- **Unchecked** → Only detected at runtime → no compile-time error.

> [!tip] Quick Memory
> Checked = Compiler says "You **must** wear helmet"  
> Unchecked = You may fall into a pothole — handle if you want

## Multi-Catch Block

Handle multiple exceptions in a **single catch** block (Java 7+).

```java
try {
    // risky code
    int a = 10 / 0;
    String s = null;
    System.out.println(s.length());
}
catch (ArithmeticException | NullPointerException e) {   // Multi-catch
    System.out.println("Handled arithmetic or null error: " + e);
}
catch (Exception e) {                                    // General catch
    System.out.println("Other error: " + e);
}
```

**Advantage**: Cleaner code when handling multiple similar exceptions.

## Finally Block

The `finally` block **always executes**, whether an exception occurs or not.  
Best used for **cleanup** (closing files, database connections, etc.).

```java
try {
    System.out.println("Trying...");
    int result = 10 / 0;
}
catch (ArithmeticException e) {
    System.out.println("Caught!");
}
finally {
    System.out.println("Finally always runs - Cleanup done!");
}
```

## try-with-resources (Auto Resource Management)

Available since Java 7. Automatically closes resources (no need for finally).

```java
import java.io.*;

public class TryWithResources {
    public static void main(String[] args) {
        try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
            String line = br.readLine();
            System.out.println(line);
        } catch (IOException e) {
            System.out.println("File error: " + e);
        }
        // br is automatically closed here - no finally needed!
    }
}
```

**Resources** that can be used: `FileReader`, `Scanner`, `Connection`, etc. (must implement `AutoCloseable`).

## throw vs throws

- **`throw`** → Manually throw an exception **inside** a method.
- **`throws`** → Declare in method signature that this method **may throw** checked exception(s).

```java
// throw keyword
static void checkAge(int age) {
    if (age < 18) {
        throw new ArithmeticException("Age must be 18 or above!");   // manual throw
    }
}

// throws keyword
static void readFile() throws IOException {     // declaring checked exception
    FileReader fr = new FileReader("abc.txt");
}
```

**throw vs throws**:

- `throw` = throw now (used with object).
- `throws` = method may throw (used in signature).

## User-Defined / Custom Exceptions

Create your own exception class for meaningful business errors.

```java
// Custom Checked Exception
class InvalidAgeException extends Exception {
    InvalidAgeException(String message) {
        super(message);
    }
}

// Usage
class CustomExceptionDemo {
    static void validateAge(int age) throws InvalidAgeException {
        if (age < 18) {
            throw new InvalidAgeException("Custom Error: You are too young!");
        }
        System.out.println("Age is valid.");
    }

    public static void main(String[] args) {
        try {
            validateAge(15);
        } catch (InvalidAgeException e) {
            System.out.println(e.getMessage());
        }
    }
}
```

> [!success] Memory Trick
>
> - Exception = Unexpected guest crashing the party.
> - try = Let's try this...
> - catch = Handle the guest politely.
> - finally = Clean up the party no matter what.
> - throw = "You! Get out now!"
> - throws = "Not my problem — caller handle it."
> - Custom = Your own bouncer with custom message.
