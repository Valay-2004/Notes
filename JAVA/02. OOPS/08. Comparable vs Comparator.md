# Comparable vs Comparator

> [!abstract] Why Sort Custom Objects?
> Java's `Collections.sort()` works easily on `String`, `Integer`, etc. (because they implement `Comparable`).  
> But for **your own classes** (Student, Employee, Product), Java doesn't know "how" to compare them.  
> → You must teach Java the sorting rule using **Comparable** (natural order inside class) or **Comparator** (external rule).  
> 
> Analogy: Sorting fruits in a basket.  
> - **Comparable** = Fruit itself knows "I am sorted by sweetness" (built-in rule).  
> - **Comparator** = You bring an external weighing machine that decides "sort by weight this time".  
> Lambda Expressions = Short, modern way to write Comparator (no extra class needed).

**Key Interfaces** (from java.lang and java.util):
- `Comparable<T>` → for natural ordering (one way only).
- `Comparator<T>` → for custom/multiple ordering (flexible).

## 1. Comparable Interface (Natural Ordering – "Inside the Class")

- Implement `Comparable<T>` in **your own class**.
- Override `compareTo(T o)` method.
- Return:
  - Negative → this object is smaller
  - Zero → equal
  - Positive → this object is bigger
- Used by `Collections.sort(list)` automatically.

**Full Code Example – Sorting Students by Marks (Natural Order)**

```java
class Student implements Comparable<Student> {   // Step 1: Implement interface
    String name;
    int marks;

    Student(String name, int marks) {
        this.name = name;
        this.marks = marks;
    }

    // Step 2: Override compareTo (decides sorting logic)
    @Override
    public int compareTo(Student other) {
        // Sort by marks (ascending)
        return this.marks - other.marks;   // or Integer.compare(this.marks, other.marks)
        
        // Alternative: sort by name (alphabetical)
        // return this.name.compareTo(other.name);
    }

    @Override
    public String toString() {
        return name + " (" + marks + ")";
    }
}

public class ComparableDemo {
    public static void main(String[] args) {
        ArrayList<Student> students = new ArrayList<>();
        students.add(new Student("Alice", 85));
        students.add(new Student("Bob", 92));
        students.add(new Student("Charlie", 78));

        System.out.println("Before: " + students);
        
        Collections.sort(students);   // Uses compareTo automatically
        
        System.out.println("After (sorted by marks): " + students);
    }
}
```

**Output:**
```
Before: [Alice (85), Bob (92), Charlie (78)]
After (sorted by marks): [Charlie (78), Alice (85), Bob (92)]
```

**Deep Points:**
- `compareTo` decides the **natural order** of your objects.
- Only **one** natural order per class.
- If you want reverse order → return `other.marks - this.marks`.

## 2. Comparator Interface (Custom Ordering – "Outside the Class")

- Does **not** implement in Student class.
- Create a **separate class** that implements `Comparator<Student>`.
- Override `compare(T o1, T o2)`.
- More powerful → multiple comparators for same class (sort by name, by age, by marks descending, etc.).
- Can be used with `Collections.sort(list, comparator)`.

**Full Code Example – Sorting Students by Name (Custom Comparator)**

```java
import java.util.Comparator;

class NameComparator implements Comparator<Student> {
    @Override
    public int compare(Student s1, Student s2) {
        return s1.name.compareTo(s2.name);   // alphabetical by name
    }
}

// In main()
public class ComparatorDemo {
    public static void main(String[] args) {
        ArrayList<Student> students = new ArrayList<>();
        students.add(new Student("Bob", 92));
        students.add(new Student("Alice", 85));
        students.add(new Student("Charlie", 78));

        System.out.println("Before: " + students);
        
        Collections.sort(students, new NameComparator());   // pass comparator
        
        System.out.println("After (sorted by name): " + students);
    }
}
```

**Modern Way: Anonymous Inner Class (Old Style)**

```java
Collections.sort(students, new Comparator<Student>() {
    @Override
    public int compare(Student s1, Student s2) {
        return s1.marks - s2.marks;
    }
});
```

## 3. Lambda Expressions (Java 8+ – Short & Clean Comparator)

Lambda = Anonymous function → replaces bulky anonymous classes.

Syntax: `(parameters) -> { body }`

**Same Example with Lambda**

```java
// Sort by marks ascending (lambda)
Collections.sort(students, (s1, s2) -> s1.marks - s2.marks);

// Sort by marks descending
Collections.sort(students, (s1, s2) -> s2.marks - s1.marks);

// Sort by name (using String's compareTo)
Collections.sort(students, (s1, s2) -> s1.name.compareTo(s2.name));

// Multiple fields (first by marks, then by name if marks equal)
Collections.sort(students, (s1, s2) -> {
    if (s1.marks != s2.marks) {
        return s1.marks - s2.marks;
    }
    return s1.name.compareTo(s2.name);
});
```

**Even Shorter with Comparator.comparing() (Java 8+ utility)**

```java
import static java.util.Comparator.comparing;

// Sort by marks
Collections.sort(students, comparing(Student::getMarks));

// Sort by name
Collections.sort(students, comparing(Student::getName));
```

## Quick Comparison Table

| Feature                  | Comparable                          | Comparator                              |
|--------------------------|-------------------------------------|-----------------------------------------|
| Where implemented        | Inside your class (`implements`)    | Separate class / lambda / anonymous     |
| Method                   | `compareTo(T o)`                    | `compare(T o1, T o2)`                   |
| Number of ways           | Only **1** natural order            | **Multiple** custom orders              |
| Flexibility              | Less (fixed inside class)           | High (can pass different comparators)   |
| When to use              | Default sorting for your object     | External / multiple sorting needs       |
| Lambda friendly          | Not directly                        | Yes (very clean)                        |
| Package                  | `java.lang`                         | `java.util`                             |

> [!tip] Memory Trick (Shash Code Style)
> - **Comparable** = "I know how to compare myself" → like a person who knows his own height.
> - **Comparator** = "Someone else decides how to compare me" → like a judge with different rules.
> - **Lambda** = "Super short note" instead of writing full essay (anonymous class).

> [!success] Real-World Use Cases
> - Sort employees by salary, joining date, or name.
> - Sort products by price, rating, or popularity.
> - Sort students for merit list, alphabetical roll call, etc.
> - Used heavily in **TreeSet**, **PriorityQueue**, and custom data structures.

> [!warning] Common Mistakes
> - Forgetting to implement `Comparable` → `Collections.sort()` throws ClassCastException.
> - Inconsistent `compareTo`/`compare` → breaks sorting (must be transitive, reflexive, etc.).
> - Using raw types with generics → avoid (use `List<Student>`).
> - Negative/positive return wrong → sorting becomes reverse or unstable.
