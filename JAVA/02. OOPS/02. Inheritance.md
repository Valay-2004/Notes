# Inheritance in Java - Quick Revision Notes

> **Core Idea** — Child class reuses code from Parent class  
> → "is-a" relationship  
> → Saves time + cleaner code

## Keyword
- Use **`extends`** to inherit  
  ```java
  class Dog extends Animal { }
  ```

## What Child Class Gets
- ✅ **`public`** members
- ✅ **`protected`** members
- ✅ **`default`** (package-private) members — **if in same package**
- ❌ **`private`** members (cannot access directly)

> Parent **cannot** access child's members  
> → Inheritance is **one direction only**

## Types of Inheritance (Supported in Java)

### 1. Single Inheritance
One parent → **one** child  
Animal → **Dog**

### 2. Multilevel Inheritance
Chain / Grandfather style  
LivingThing → **Human** → **Employee**

### 3. Hierarchical Inheritance
One parent → **many** children  
Vehicle  
├─ **Car**  
├─ **Bike**  
└─ **Truck**

### 4. Hybrid Inheritance
Combination of above (e.g. Hierarchical + Multilevel)  
→ Allowed (as long as no **multiple** class inheritance)

### 5. Multiple Inheritance
One child → **many** parents  
→ **NOT allowed** for **classes** in Java

> **Reason**: **Diamond Problem**  
> Father.snore() + Mother.snore() → Child confused which one to call?

```text
     Father
      /  \
     /    \
   Child   Mother
      \    /
       \  /
      Diamond
```

## Solution for Multiple Inheritance
Use **Interfaces**  
→ A class can **`implements`** many interfaces  
→ No diamond problem (interfaces had no implementation before Java 8)

## Important Default Behavior
Every class **automatically** inherits from **`Object`**  
(even if you don't write `extends Object`)

```java
class MyClass { }          // actually → class MyClass extends Object { }
```

→ Free methods you get:
- `toString()`
- `equals()`
- `hashCode()`
- `getClass()`
- `wait()`, `notify()`, `notifyAll()`

## Quick Summary Table

| Type               | Allowed in Java (Classes)? | Example                     | Notes                          |
|--------------------|----------------------------|-----------------------------|--------------------------------|
| Single             | Yes                        | Animal → Dog                | Most common                    |
| Multilevel         | Yes                        | A → B → C                   | Chain inheritance              |
| Hierarchical       | Yes                        | Animal → Dog, Cat, Cow      | One parent, many children      |
| Multiple           | **No**                     | —                           | Diamond problem                |
| Hybrid             | Yes                        | Mix of above                | As long as no multiple classes |
| Multiple via Interfaces | Yes                   | class Dog implements Pet, Mammal | Safe & recommended        |

> **Memory Trick**  
> **Yes** → Single + Multi**level** + **Hierarchical** + **Hybrid**  
> **No**  → Multiple (for classes)  
> **Yes** → Multiple via **interfaces**
