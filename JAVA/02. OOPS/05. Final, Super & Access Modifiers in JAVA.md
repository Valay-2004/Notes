# Final, super & Access Modifiers in Java - Quick Revision Notes

These three concepts strengthen **encapsulation**, control **inheritance**, and help in **parent-child communication**.

## 1. Access Modifiers (Visibility Control)

Access modifiers decide **who can see/use** variables, methods, constructors, or classes.

| Modifier                 | Same Class | Same Package (non-subclass) | Subclass (different package) | Other classes (different package) | Best Use Case                          |
| ------------------------ | ---------- | --------------------------- | ---------------------------- | --------------------------------- | -------------------------------------- |
| **public**               | Yes        | Yes                         | Yes                          | Yes                               | API methods, widely shared             |
| **protected**            | Yes        | Yes                         | Yes                          | No                                | Inheritance-friendly (subclasses only) |
| **default** (no keyword) | Yes        | Yes                         | No                           | No                                | Package-level only                     |
| **private**              | Yes        | No                          | No                           | No                                | Hide internal details (encapsulation)  |

> [!important] Overriding Rule for Access Modifiers
> Child method's access level must be **≥** parent's:
>
> - private → cannot override (not inherited)
> - default → child can use default / protected / public
> - protected → child can use protected / public
> - public → child must keep public

```java
class Parent {
    protected void show() {          // protected
        System.out.println("Parent show");
    }
}

class Child extends Parent {
    @Override
    public void show() {             // allowed: public ≥ protected
        System.out.println("Child show");
    }
}
```

## 2. final Keyword (Immutability & Restriction)

`final` prevents changes — value once set, cannot change; method cannot override; class cannot extend.

- **final variable** → constant (assign once, usually at declaration or in constructor)
- **final method** → cannot override
- **final class** → cannot extend (e.g., String, Integer are final)

```java
// final variable (constant)
final double PI = 3.14159;
// PI = 3.14;  // ERROR: cannot reassign

// final method
class Vehicle {
    final void fuelType() {
        System.out.println("Petrol/Diesel");
    }
}

class ElectricCar extends Vehicle {
    // void fuelType() { ... }  // ERROR: cannot override final method
}

// final class
final class Constants { }
// class MyConstants extends Constants { }  // ERROR: cannot extend final class
```

> [!tip] Memory Trick
> final = "final decision" — no more changes allowed at that level.

## 3. super Keyword (Access Parent from Child)

Used inside child class to refer to **parent's members** (variables, methods, constructors).

Three main uses:

1. **super()** – Call parent constructor (must be **first line** in child constructor)
2. **super.method()** – Call parent method (useful after overriding)
3. **super.variable** – Access parent variable (when child has same name)

```java
class Animal {
    String color = "Brown";
    Animal() {
        System.out.println("Animal created");
    }
    void eat() {
        System.out.println("Animal eats");
    }
}

class Dog extends Animal {
    String color = "Black";

    Dog() {
        super();                    // calls parent constructor
        // or super("some param") if parent has parameterized ctor
    }

    @Override
    void eat() {
        super.eat();                // calls parent's eat()
        System.out.println("Dog eats bones");
    }

    void showColor() {
        System.out.println(color);       // Black (child)
        System.out.println(super.color); // Brown (parent)
    }
}
```

> [!example] In main():
>
> ```java
> Dog d = new Dog();
> d.eat();          // Output: Animal eats \n Dog eats bones
> d.showColor();    // Black \n Brown
> ```

> [!warning] Common Mistakes
>
> - super() not first line in constructor → compile error
> - Using super in static method → error (super is for instance context)
> - Trying to override private/final/static methods → won't work

## Quick Summary Table

| Concept          | Purpose                           | Key Restriction/Rule                    | Example Keyword/Usage               |
| ---------------- | --------------------------------- | --------------------------------------- | ----------------------------------- |
| Access Modifiers | Control visibility                | Child cannot reduce parent's visibility | public, protected, default, private |
| final            | Prevent change/override/extension | Assign once; no override; no extend     | final int x=10; final void run()    |
| super            | Access parent members from child  | First in ctor; instance context only    | super(); super.eat(); super.color   |

> [!success] One-line Memory Trick
>
> - **Access modifiers** = Who can see me?
> - **final** = No changes allowed!
> - **super** = "Hey parent, let me borrow your stuff"
