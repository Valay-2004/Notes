## Two Ways to Achieve Abstraction

### 1. Abstract Class (Partial Abstraction – 0% to 100%)

- Use keyword **`abstract`**
- Can have both **abstract methods** (no body) + **concrete methods** (with body)
- Cannot create object: `new AbstractClass()` → ❌
- Child class **must** implement all abstract methods (or become abstract itself)

```java
// Abstract class example
abstract class Vehicle {                // incomplete class
    abstract void start();              // no body → must override

    void stop() {                       // concrete method (optional)
        System.out.println("Vehicle stopped.");
    }
}

class Bike extends Vehicle {
    @Override
    void start() {
        System.out.println("Bike started with kick.");
    }
}

public class Main {
    public static void main(String[] args) {
        Vehicle v = new Bike();         // Upcasting – allowed
        v.start();                      // Bike started with kick.
        v.stop();                       // Vehicle stopped.
        // Vehicle v2 = new Vehicle();  // ERROR – cannot instantiate
    }
}
```

### 2. Interface (100% Abstraction – before Java 8)

- Use keyword **`interface`**
- All methods are **public abstract** by default (no need to write abstract)
- No concrete methods (until Java 8 – default & static methods added later)
- Class can **`implements`** multiple interfaces → solves multiple inheritance issue
- 100% abstraction (pure contract – no implementation)

```java
// Interface – 100% abstraction
interface Animal {
    void eat();                         // public abstract by default
    void sleep();
}

class Dog implements Animal {
    @Override
    public void eat() {
        System.out.println("Dog eats bones.");
    }

    @Override
    public void sleep() {
        System.out.println("Dog sleeps.");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a = new Dog();           // reference of interface
        a.eat();                        // Dog eats bones.
        a.sleep();                      // Dog sleeps.
    }
}
```

> [!important] Key Differences – Quick Table

| Feature           | Abstract Class                | Interface (pre-Java 8)                 |
| ----------------- | ----------------------------- | -------------------------------------- |
| Keyword           | `abstract class`              | `interface`                            |
| Methods           | Abstract + Concrete           | Only abstract (100%)                   |
| Variables         | Can have instance/static vars | Only `public static final` (constants) |
| Inheritance       | `extends` (single)            | `implements` (multiple allowed)        |
| Instantiation     | Cannot create object          | Cannot create object                   |
| Abstraction Level | Partial (0–100%)              | 100%                                   |
| Use when...       | Common code/behavior to share | Pure contract / multiple inheritance   |

> [!example] Real-world Analogy (Navin-Style)
> Remote control of TV → you press "Power", "Volume" (functionality shown)  
> You don't know internal circuits/wires (implementation hidden)  
> → That's **Abstraction**!

> [!success] Memory Trick
>
> - Abstract class = **partial blueprint** (some ready parts + some to-dos)
> - Interface = **full contract** (only to-dos, no ready parts) → 100% abstraction  
>   → "Interface = 100% promise, no delivery yet"
