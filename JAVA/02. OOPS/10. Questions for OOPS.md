# 🚀 OOPS Interview Questions & Answers

A curated collection of top interview questions for Object-Oriented Programming, categorized by core concepts.

---

## 1. Fundamentals (Classes & Objects)

**Q1: What is the difference between a Class and an Object?**

- **Class:** A blueprint, template, or logical entity. It doesn't occupy memory.
- **Object:** An instance of a class, a physical entity. It occupies memory when created.

**Q2: What is a Constructor? Can it be inherited?**

- A constructor is a special method used to initialize objects. It has the same name as the class and no return type.
- **No**, constructors cannot be inherited, but they are called during the parent-child initialization process.

**Q3: Can a class be static in Java?**

- A top-level class cannot be static. Only **nested classes** can be static.

**Q4: What is the purpose of the `final` keyword?**

- **Variable:** Becomes a constant (cannot be reassigned).
- **Method:** Cannot be overridden.
- **Class:** Cannot be inherited.

**Q5: Difference between a Static Method and an Instance Method?**

- **Static Method:** Belongs to the class; can be called without an object. Cannot access instance variables.
- **Instance Method:** Belongs to an object; requires an instance to be called. Can access both static and instance variables.

## 2. Encapsulation

**Q1: How do you achieve Encapsulation in Java?**

- By making fields `private` and providing `public` getter and setter methods.

**Q2: What is the benefit of Encapsulation?**

- Data hiding, security, validation, and the ability to change internal implementation without affecting external code.

**Q3: What is "Defensive Copying"?**

- Returning a copy of a mutable private field (like a List or Date) instead of the original reference, preventing external modification of private data.

**Q4: Difference between Encapsulation and Abstraction?**

- **Encapsulation:** Hides **data** (implementation details) using access modifiers. It's about "How" to hide.
- **Abstraction:** Hides **complexity** (functionality) using interfaces/abstract classes. It's about "What" to hide.

**Q5: Can we have a private constructor?**

- **Yes.** It's used in the **Singleton Design Pattern** to prevent other classes from instantiating the class directly.

## 3. Inheritance

**Q1: Why doesn't Java support multiple inheritance with classes?**

- To avoid complexity and ambiguity known as the **Diamond Problem**. Interfaces provide a safer alternative.

**Q2: What is the use of the `super` keyword?**

- To call the parent class constructor, access parent methods, or refer to parent variables.

**Q3: What is the difference between IS-A and HAS-A relationships?**

- **IS-A:** Inheritance (e.g., Dog is an Animal).
- **HAS-A:** Association/Composition (e.g., Car has an Engine).

**Q4: What is the Liskov Substitution Principle (LSP)?**

- It states that objects of a superclass should be replaceable with objects of a subclass without affecting the program's correctness.

**Q5: Why is the `Object` class considered the root of all classes?**

- It provides common behaviors (like `toString()`, `equals()`, `hashCode()`) that every Java object needs, ensuring a consistent baseline for all types.

## 4. Polymorphism

**Q1: Method Overloading vs. Method Overriding?**
| Feature | Overloading | Overriding |
|---------|-------------|------------|
| Binding | Compile-time (Static) | Runtime (Dynamic) |
| Location| Same class | Different classes (Inheritance) |
| Params | Must differ | Must be same |

**Q2: Can we override a static method?**

- **No.** Static methods are bound to the class, not the object. If you redefine it in a child class, it is called **Method Hiding**.

**Q3: What is "Covariant Return Type"?**

- When an overriding method in a subclass returns a more specific type than the method in the superclass.

**Q4: What is "Dynamic Binding" (Late Binding)?**

- It refers to the process where the call to an overridden method is resolved at **runtime** rather than compile-time, based on the actual object type.

**Q5: Does Java support Operator Overloading?**

- **No.** To keep the language simple and less error-prone. The only exception is the `+` operator, which is overloaded for String concatenation.

## 5. Abstraction

**Q1: Interface vs. Abstract Class?**

- **Abstract Class:** Can have instance variables, constructors, and both abstract/concrete methods.
- **Interface:** Only abstract methods (pre-Java 8), no instance variables, supports multiple inheritance.

**Q2: Can an Abstract Class have a constructor?**

- **Yes.** It's used for initialization during constructor chaining when a subclass is instantiated.

**Q3: Can we instantiate an interface?**

- **No.** You must implement it through a class or an anonymous inner class.

**Q4: Can an abstract class implement an interface?**

- **Yes.** It doesn't even have to provide implementations for the interface methods; it can delegate that responsibility to its first concrete subclass.

**Q5: What is a Functional Interface?**

- An interface that contains **exactly one** abstract method. They are used extensively with **Lambda Expressions** (e.g., `Runnable`, `Comparator`).

## 6. Advanced Topics

**Q1: What is an Anonymous Inner Class?**

- A class without a name, used for one-time implementation of an interface or abstract class.

**Q2: What is "Type Erasure" in Generics?**

- The process where the compiler removes generic type information at compile-time to ensure backward compatibility.

**Q3: Difference between Composition and Aggregation?**

- **Composition:** Strong "Part-of" (Engine dies with Car).
- **Aggregation:** Weak "Has-a" (Student lives without University).

**Q4: Shallow Copy vs. Deep Copy?**

- **Shallow Copy:** Copies the object but shares the same references for internal objects.
- **Deep Copy:** Creates a completely independent copy of the object and all its internal objects.

**Q5: What is the Diamond Problem?**

- A conflict in multiple inheritance where a class inherits from two classes that both inherit from a single parent, leading to ambiguity in which parent's method to use. Java avoids this with Interfaces.

## 💡 Expert Tips for Interviews

1. **Always use examples:** Don't just define; give a real-world scenario (e.g., "Think of a BankAccount for Encapsulation").
2. **Mention SOLID:** If asked about OOP design, briefly mention SOLID principles to show advanced knowledge.
3. **Be specific about Java versions:** Mention that Interfaces can have `default` methods since Java 8.
